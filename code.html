Repository standard

Public code should be inspectable, reproducible, and honest about its limits.

Documentation

Purpose, scientific context, inputs, outputs, installation, dependencies, and example results.

Reproducibility

Versioned dependencies, deterministic settings where appropriate, test data, and transparent preprocessing.

Reuse

Licence, citation guidance, file structure, and clear separation of private data from public code.

Limitations

Known failure modes, assumptions, statistical caveats, and boundaries of valid interpretation.
